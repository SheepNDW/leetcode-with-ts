## 1526. Minimum Number of Increments on Subarrays to Form a Target Array

可以將 target 想像成一條「連續起伏的山脈」：

* 每一次操作等同於「壘土造山」。
* 在上坡時，我們必須不斷新增操作去讓高度上升。
* 在下坡時，則不需要額外操作，因為之前的造山已經覆蓋到那段區域。

每次上坡就代表需要「重新造山」，下坡代表暫時可以「休息」，因為那段已經被先前的操作支撐起來了。

舉例來說，對於 `target = [1, 3, 4, 2]`：

| 位置變化   | 說明                                                |
| ---------- | --------------------------------------------------- |
| 0 -> 1(+1) | 自 0 到 1，需要 1 次操作來將高度從 0 提升到 1。     |
| 1 -> 3(+2) | 自 1 到 3，需要額外 2 次操作來將高度從 1 提升到 3。 |
| 3 -> 4(+1) | 自 3 到 4，需要額外 1 次操作來將高度從 3 提升到 4。 |
| 4 -> 2(-2) | 下降，不需要額外操作。                              |

總操作次數為 1 + 2 + 1 = 4。

如果我們在 `target` 再擴充成 `[1, 3, 4, 2, 4, 1]`：

| 位置變化   | 說明                                                |
| ---------- | --------------------------------------------------- |
| 2 -> 4(+2) | 自 2 到 4，需要額外 2 次操作來將高度從 2 提升到 4。 |
| 4 -> 1(-3) | 下降，不需要額外操作。                              |

總操作次數為 1 + 2 + 1 + 2 = 6。

我們把每次從下坡轉成上坡時都當成一個新的「起點」，並將每次上坡的增量加總起來，即可得到最終答案。
