## 2211. Count Collisions on a Road

這題除了可以用 stack 從左到右模擬碰撞過程外，還可以直接用貪心的思想來解：

只要左邊有一台靜止 `'S'` 或往右 `'R'` 的車，那麼它右邊所有往左 `'L'` 的車最後都一定會撞。
反之，只要右邊有一台靜止 `'S'` 或往左 `'L'` 的車，那麼它左邊所有往右 `'R'` 的車最後也一定會撞。

根據這個觀察，我們可以將整條道路分成三部分：

- 最左側那一串純 `'L'` → 無論如何都不會撞
- 最右側那一串純 `'R'` → 無論如何都不會撞
- 中間的部分 → 只要是 `'L'` 或 `'R'` 都一定會發生碰撞（或參與碰撞），`'S'` 不會增加碰撞次數，但會停止其他車的移動

因此我們可以直接做兩次遍歷來計算：

1. 從左到右遍歷，找到中間區間所有往左 `'L'` 車的數量：
   - 用一個 flag 標記是否已經出現過 `'S'` 或 `'R'`，一旦出現後，之後遇到的所有 `'L'` 都會增加碰撞次數
   - 一但進入中間區間後，每個遇到的 `'L'` 都表示這台車會撞，碰撞次數加一

2. 從右到左遍歷，找到中間區間所有往右 `'R'` 車的數量：
   - 同理，用一個 flag 標記是否已經出現過 `'S'` 或 `'L'`，一旦出現後，之後遇到的所有 `'R'` 都會增加碰撞次數
   - 一但進入中間區間後，每個遇到的 `'R'` 都表示這台車會撞，碰撞次數加一

最後答案就會是兩次遍歷中計算出的碰撞次數總和。
