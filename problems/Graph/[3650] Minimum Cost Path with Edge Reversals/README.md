## 3650. Minimum Cost Path with Edge Reversals

在這道題目中，我們需要找出從節點 0 到節點 n−1 的最短路徑。題目允許在抵達某個節點時，將一條指向該節點的邊反轉，並立刻沿反轉後的方向走出，其成本為原邊權重的兩倍。由於這個操作只影響當下的一次移動，且不會永久改變圖的結構，因此可以在建圖階段直接將其轉換為等價的加權邊。

具體作法是：對於每條原始邊 `[u, v, w]`，在圖中加入兩條邊：
- `u → v`，權重為 `w`（表示不反轉邊的情況）
- `v → u`，權重為 `2w`（表示反轉邊並立刻走的情況）

透過這樣的建圖方式，所有合法的移動方式都被轉換為固定成本的邊，使得問題能夠直接套用標準的 Dijkstra 模板來求解最短路徑。

### 實作細節：

可以使用 min-heap 來實作 Dijkstra 演算法。每一步都取出所有尚未確定最短距離的節點中，暫定距離最小者，並嘗試更新其相鄰節點的距離。由於同一個節點可能會被多次加入 heap，因此需要在取出時忽略已處理或過期的狀態。
