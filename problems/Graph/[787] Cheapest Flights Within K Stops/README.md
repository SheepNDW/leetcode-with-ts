## 787. Cheapest Flights Within K Stops

題目給了一張有向加權圖，用以描述城市之間的航班資訊 (節點代表城市，邊代表航班，邊的權重代表票價)。目標是在限制中轉次數 `k` 的情況下，找出從起點城市 `src` 飛往終點城市 `dst` 的最低票價。

首先要先明確「中轉次數」的定義：如果從城市 A 飛往城市 B 再飛往城市 C，這樣的行程中轉次數為 1（因為有一次停留在 B）。因此，如果 `k = 0`，表示只能搭乘直飛航班。所以 `航班數 = 中轉次數 + 1`。


### Dijkstra

利用 Dijkstra 求最短權重的路徑，可以簡單理解成基於優先佇列的 BFS。從起點 `src` 開始不停地 BFS，但是佇列中優先彈出的永遠是權重最小的路徑（所到達的點）。這樣任何彈出的點。如果是第一次到達的話，那麼它一定是從起點出發到該點的最短路徑。所以當我們彈出終點 `dst` 時，就可以直接回傳該路徑的最短代價。

不過這題有個不同之處是，我們需要限制中轉次數 `k`。對於任何的中轉點（非終點），最短路徑不一定是最佳方案，因為可能會超過中轉次數的限制。因此我們需要在狀態中加入「目前中轉次數」這個資訊。也就是說，我們需要追蹤三個資訊：目前所在的城市、目前累積的票價、目前的中轉次數。對於不同的中轉次數，我們可能會有不同的最短路徑，所以都必須加入優先佇列中。當然如果之前曾經以相同的中轉次數到達過該城市，我們只保留票價較低的路徑，所以在實作時還需要準備一個 `costs[city][stops]` 來記錄每個城市在不同中轉次數下的最低票價。
