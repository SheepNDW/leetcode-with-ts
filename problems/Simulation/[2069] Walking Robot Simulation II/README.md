## 2069. Walking Robot Simulation II

這是一道 simulation 題，機器人沿矩形邊界行走，需要模擬位置與方向的變化。

### 邊界參數

- 格點平面範圍：`(0, 0)` 到 `(width-1, height-1)`
- 初始位置：`(0, 0)`，面朝東（East）
- 行走順序：East → North → West → South → 循環（沿邊框逆時針）
- 一圈總步數：`total = 2 * (width + height) - 4`

### 解題思路

1. 計算當前方向還能走幾步：根據目前方向與位置，算出碰到下一個角落前的剩餘步數 `remain`。
   若 `remain >= num`，直接走完，結束。

2. 碰到角落還有剩餘步數：走到角落後轉向（順序 East → North → West → South），
   扣掉 `remain` 後繼續以相同邏輯處理剩餘步數。

3. 利用週期性剪枝：每次抵達角落並轉向後，立即對剩餘步數取模：
   `num %= total`  這樣可以消除「繞完整圈但回到同一位置」的重複計算，確保 while 迴圈最多只執行 4 次（對應四個角落），整體達到 O(1)。
4. 特例：停在角落且步數恰為 total 的倍數：
   若取模後 `num === 0`，且當前位置恰好是角落，
   代表機器人走完整數圈後停在角落——此時方向**不應該**換成下一個方向。
   由於步驟 2 已經提前執行了轉向，需要將方向倒退 90 度來修正：
   `this.d = (this.d + 3) % 4  // 等同於 -1 mod 4`


#### 為何需要倒退而非不轉？
流程上「抵達角落」和「轉向」是同一步完成的，取模後才發現 `num === 0`，
方向已經多轉了一次，所以用 `+3` 來回退，而不是在轉向前多加一層判斷。
