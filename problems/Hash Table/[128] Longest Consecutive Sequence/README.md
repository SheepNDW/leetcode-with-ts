## 128. Longest Consecutive Sequence

本題要求在 `nums` 陣列中找出最長的連續整數序列長度，並且要求時間複雜度為 `O(n)`。這代表不能單純使用 sorting 後在遍歷求解，因為 sorting 的時間複雜度為 `O(n log n)`。

有一個很巧妙的解法是利用 Set 來解。首先將所有元素放入一個 `set` 中，這樣可以讓我們在 `O(1)` 的時間內檢查某個數字是否存在於陣列中，同時也去除了重複的元素。

接下來遍歷 `set` 中的每個元素，對於每個元素 `num`，我們檢查 `num - 1` 是否存在於 `set` 中。如果不存在，這表示 `num` 是一個序列的起點。然後我們從 `num` 開始，不斷檢查 `num + 1`, `num + 2`, ... 是否存在於 `set` 中，直到找不到為止。這樣我們就能計算出以 `num` 為起點的連續序列長度。

最後答案就是所有起點所對應的連續序列長度中的最大值。
