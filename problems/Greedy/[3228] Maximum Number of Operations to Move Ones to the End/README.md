## 3228. Maximum Number of Operations to Move Ones to the End


我們可以對字串 `s` 執行以下操作：

- 選擇某個 index `i`，滿足 `s[i] == '1'` 且 `s[i+1] == '0'`（也就是一個 `"10"`）。
- 接著將這個 `'1'` 一路向右滑動，穿過後面的連續 `0...0`，直到：
  - 被下一個 `'1'` 阻擋，或
  - 到達字串末尾。
- 無論這過程滑過多少個 `0`，都僅算一次操作。

目標是：在不能再做任何操作之前，**最多能執行多少次操作**。

最終字串一定變成：`000...0111...1`。

#### 思路：

可以留意到每次操作只是把某個 `1` 向右滑到下一個 `1` 前面，因此所有 `1` 的最終排列順序是固定不變的。

每個 `'1'` 在整個過程中會被「選中」幾次？ 

一次操作本質上是：某個 `1` 在遇到一段連續的 `0` 區塊時，可以從 `10` 出發，滑過那整段 `0...0`，完成一次操作。

注意：不管一段 `0` 裡有幾個 `0`，這段區塊只會讓一個 `1` **多一次**操作機會。

因此，每一段連續的 `0` 區塊，都會讓左邊所有的 `1` 各貢獻一次操作。

我們可以遍歷字串，記錄目前遇到的 `1` 數量 `ones`，然後每當遇到一段 `0` 區塊時，就把 `ones` 加到答案中。
