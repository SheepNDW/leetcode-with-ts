## 3070. Count Submatrices with Top-Left Element and Sum Less Than k

題目要求我們在一個二維矩陣中，計算有多少個子矩陣同時滿足以下兩個條件：

* 條件一：包含左上角元素
子矩陣的左上角必須是整個矩陣的 `[0][0]` 位置（即第 0 列、第 0 行）。
這代表每個合法的子矩陣，都是從原點 (0, 0) 延伸到某個位置 (i, j) 的矩形區域。

* 條件二：元素總和 `≤ k`
該子矩陣內所有元素的加總必須 `≤ k`。

我們定義一個二維 prefix，`prefix[i][j]`，表示從矩陣的左上角 (0, 0) 到位置 (i, j) 的子矩陣內所有元素的總和。那麼要計算 `prefix[i][j]` 時，我們會想到把上面那塊矩形和左邊那塊矩形的總和加起來，然後再加上自己 `grid[i][j]` 的值，不過這樣會重複加上左上角那塊矩形，所以我們需要減去一次：

```ts
prefix[i][j] = grid[i][j]
             + prefix[i-1][j]    // 上方矩形
             + prefix[i][j-1]    // 左方矩形
             - prefix[i-1][j-1]  // 重複的左上角矩形
```

用圖來理解：

```
┌─────────────┬───┐
│             │   │
│  [i-1][j-1] │   │  ← prefix[i-1][j]（上方）
│             │   │
├─────────────┼───┤
│             │ x │  ← grid[i][j]（自己）
└─────────────┴───┘
     ↑
  prefix[i][j-1]（左方）
```

我們只要建構出這個 prefix 矩陣，就可以在 O(1) 的時間內計算出任何子矩陣的總和。對於每個位置 (i, j)，我們檢查 `prefix[i][j]` 是否 `≤ k`，如果是的話，我們就把計數器加一。
